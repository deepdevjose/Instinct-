Quiero construir una webapp 3D llamada INSTINCT(), un videojuego educativo para aprender Python desde cero mediante una historia narrativa de una serpiente.

Objetivo general:
Crear un prototipo funcional en web donde el jugador controla/programa una serpiente 3D usando código estilo Python. El juego debe integrar un roadmap educativo de Python desde nivel cero hasta conceptos avanzados, dentro de una historia de vida completa: la serpiente nace de un huevo, aprende a moverse, cazar, sobrevivir, mudar su piel, proteger crías, enseñar comportamiento heredado y finalmente muere de vejez.

Stack recomendado:
- React
- Vite
- TypeScript
- React Three Fiber
- Three.js
- TailwindCSS
- Zustand para estado global
- Monaco Editor para el editor de código
- Pyodide o un intérprete simulado inicial para ejecutar comandos tipo Python

Requisitos principales:
1. Crear una webapp 3D con layout dividido:
   - Mundo 3D principal a la izquierda.
   - Panel derecho con:
     - título del nivel,
     - objetivo,
     - explicación breve del concepto de Python,
     - editor de código,
     - botón Run,

     - progreso del roadmap.

2. Crear un mundo 3D inicial:
   - Campo natural con pasto, piedras, árboles bajos, niebla ligera.
   - Serpiente animada de forma sencilla.
   - Huevo inicial.
   - Depredadores/cazadores básicos.
   - Cámara isométrica o tercera persona.

3. Implementar la lógica base del juego:
   - Entidad Snake con posición, vida, energía, veneno, edad, estado y etapa de vida.
   - Entidades Predator/Hunter con posición, rango de detección y comportamiento simple.
   - Sistema de niveles.
   - Sistema de ejecución de código.
   - Validación de objetivos por nivel.
   - Consola de feedback.

Mecánica de fallo, muerte y checkpoints:

1. Si el jugador ejecuta código incorrecto:
   - La serpiente debe morir o fallar narrativamente según el nivel.
   - Mostrar una animación corta de muerte/fallo.
   - Mostrar mensaje en consola explicando el error.
   - Reiniciar el nivel desde el último checkpoint.

2. Cada nivel debe tener:
   - checkpointState
   - attempts
   - maxAttempts = 3
   - failureNarrative
   - hintLevel1
   - hintLevel2
   - hintLevel3

3. Si el jugador falla 1 vez:
   - Reiniciar el mismo nivel.
   - Mostrar pista ligera.

4. Si falla 2 veces:
   - Reiniciar el mismo nivel.
   - Mostrar pista más directa.

5. Si falla 3 veces:
   - Mostrar pista explícita.
   - Reiniciar el mismo nivel por última vez.

6. Si falla más de 3 veces:
   - Regresar automáticamente al nivel anterior.
   - Mostrar mensaje narrativo:
     “El instinto aún no está listo. Regresa, aprende y vuelve más fuerte.”

7. Si el jugador está en el nivel 1 y falla más de 3 veces:
   - No puede regresar más.
   - Se reinicia el nivel 1 con una guía más clara.

8. El sistema debe guardar:
   - nivel actual,
   - intentos por nivel,
   - checkpoints,
   - progreso desbloqueado.

9. Implementar funciones en el store:
   - runCode()
   - handleSuccess()
   - handleFailure()
   - resetToCheckpoint()
   - goToPreviousLevel()
   - incrementAttempts()

10. La muerte no debe sentirse como castigo injusto.
    Debe sentirse como parte del aprendizaje:
    error → feedback → intento → mejora.

Y en la estructura del nivel cambia esto:

type Level = {
  id: number;
  title: string;
  act: string;
  narrative: string;
  pythonConcept: string;
  objective: string;
  starterCode: string;
  expectedPatterns: string[];
  successCondition: string;
  unlockedCommands: string[];

  checkpointState: {
    snakePosition: [number, number, number];
    snakeHealth: number;
    snakeEnergy: number;
    snakeVenom: number;
    predators: any[];
    children?: any[];
  };

  maxAttempts: number;
  failureNarrative: string;
  hints: {
    first: string;
    second: string;
    third: string;
  };
};

Esto le mete tensión real al juego: programar bien = sobrevives, programar mal = mueres y aprendes.
